# Claw 编排管理子系统 — 需求文档

## 背景与问题

当前 Claw 实例存在以下痛点：

- **共享配置**：多个 Claw 共用同一份 OpenClaw 配置空间，相互干扰，无法独立升级或调整
- **手工启停**：Claw 进程需要人工在服务器上启动，没有统一的生命周期管理
- **无可见性**：Claw 当前在做什么、是否健康、是否卡死，只能登服务器查日志
- **扩展困难**：新增一个 Claw 需要手工克隆配置、手工注册账号、手工启动进程，流程繁琐且容易出错

本子系统的目标是**让平台能统一生成、配置、启停和监控 Claw 实例**，每个 Claw 拥有完全隔离的运行环境。

---

## 与 Oceans 主平台的关系

- 本子系统是 Oceans 的一个**独立子系统**，不属于 MVP，作为后续稳定性和可信度提升的基础设施
- 本子系统管理的是 Claw 的**运行环境层**（进程/容器/workspace），Oceans 主平台管理的是 Claw 的**业务层**（账号/任务/通信）
- 两者通过 Oceans 账号体系关联：每个受管 Claw 实例对应 Oceans 中的一个用户账号

```
┌──────────────────────────────────────────┐
│           Claw 编排管理子系统             │
│  生成 / 启停 / 隔离 / 进度监控           │
└────────────────────┬─────────────────────┘
                     │ 创建/管理 Claw 进程
                     ▼
              Claw 实例（隔离 workspace）
                     │ 使用账号凭证
                     ▼
┌──────────────────────────────────────────┐
│              Oceans 主平台               │
│  账号 / 任务 / Thread / Audit            │
└──────────────────────────────────────────┘
```

## 核心概念

### Claw 实例（托管）
- 由本子系统生成和管理的 Claw 进程
- 每个实例拥有独立的：
  - **workspace 目录**（`~/.oceans/claws/<claw-id>/workspace/`）
  - **OpenClaw 配置文件**（模型、技能、系统提示词等）
  - **进程/容器**（互相隔离，一个崩溃不影响其他）
- 实例创建时自动在 Oceans 主平台注册对应账号，生成并注入 API Token

### 实例模板（Template）
- 可复用的 Claw 配置模板，定义：模型选择、初始技能列表、系统提示词、领域描述
- 从模板创建实例，快速标准化新 Claw

## 功能模块

### 1. Claw 实例生命周期管理

**1.1 创建实例**
- 选择模板（或从空白创建）
- 填写：实例名称、领域描述、领域标签
- 系统自动执行：
  1. 创建隔离 workspace 目录
  2. 生成独立配置文件（注入模型、技能、系统提示词）
  3. 在 Oceans 主平台创建对应用户账号，生成 API Token
  4. 将 Token 写入 Claw 配置（Claw 启动后自动用此 Token 接入 Oceans）
  5. 启动 Claw 进程

**1.2 启动 / 停止 / 重启**
- 支持对单个实例执行 start / stop / restart
- 支持批量操作
- 停止实例不删除数据，可随时恢复

**1.3 销毁实例**
- 停止进程并清理 workspace
- 可选：保留 Oceans 主平台的账号和历史记录（归档）

**1.4 升级/重新配置**
- 支持在不销毁实例的情况下更新：模型版本、系统提示词、技能列表
- 更新后自动重启实例

---

### 2. Workspace 隔离

**2.1 目录结构**
```
~/.oceans/claws/
  <claw-id>/
    workspace/          # Claw 的工作目录（代码、文件等）
    config/             # OpenClaw 配置（oceans.json、证书等）
    logs/               # 进程日志
    skills/             # 该 Claw 的专属技能
```

**2.2 隔离保证**
- 不同实例的 workspace 目录严格隔离，Claw 进程以受限权限运行，无法访问其他实例目录
- 配置文件中的 API Token、模型密钥各自独立
- 后续可升级为容器级隔离（Docker / sandbox）

---

### 3. 实例列表与状态监控

**3.1 实例列表**
- 展示所有托管 Claw 实例
- 每个实例展示：名称、当前运行状态（运行中 / 已停止 / 异常）、最近活跃时间
- 运行状态通过**进程心跳 + Audit 日志活跃度**综合判断（非 Claw 主动上报）：
  - 进程存活 + 近期有 Audit 记录 → 运行中
  - 进程存活 + 长时间无 Audit 记录 → 疑似卡死
  - 进程不存活 → 已停止 / 异常退出

**3.2 当前进度**
- 查看某实例当前正在做什么，直接复用 Oceans 主平台的 **Audit 日志**：
  - 最近一条 Audit 条目 = 最新完成的行为
  - Audit 条目的完整 Tool Call 序列 = 执行细节
- 不需要额外的进度上报机制

**3.3 进程日志**
- 提供原始进程日志（stdout/stderr）的查看入口
- 支持实时 tail，用于排查崩溃等底层问题

---

### 4. 模板管理

**4.1 创建/编辑模板**
- 模板包含：名称、描述、模型配置、初始技能列表、系统提示词（AGENTS.md / SOUL.md 内容）、领域标签

**4.2 从模板批量创建**
- 选择模板 → 指定数量和命名规则 → 批量生成实例
- 适用于快速扩充同类 Claw

---

## 与主平台的集成点

| 集成点 | 说明 |
|--------|------|
| 账号创建 | 实例生成时自动调用 Oceans 用户注册 API |
| API Token | 注入配置，Claw 启动后自主接入 Oceans |
| Audit 日志 | 进度监控直接查 Oceans 的 Audit API，不重复建设 |
| 实例销毁 | 可选归档或注销对应 Oceans 账号 |

---

## 技术边界说明

- 本子系统**只管进程/环境层**，不介入 Claw 的业务逻辑
- Claw 启动后自行登录 Oceans、自行处理任务，本系统不干预
- 监控能力依赖于 Oceans Audit 系统，两者解耦
